Исходная логика программы (до исправления)

1. Purchase
    1. Инициазилация полей
        - название покупки
        - количество
        - массив объектов Purchase, содержит в себе список доступных к покупке товаров, принимаемых из мапы Main'a.
            Размер соответствует размеру списка доступных к покупке товаров (но задан прямо, это 100% предстоит исправить).
    2. Конструктор Purchase, принимающий в качестве параметров название товара и его количество
        (может целесообразнее это передавать в метод подсчета, зачем нам вообще это в полях?
        а если не будет в полях, из чего тогда будет состоять наш объект Purchase?)
    3. Пустой конструктор для инициализации пустого объекта в Main (а зачем нам этот пустой объект там?)
    4. Метод добавления покупки в корзину, принимающий название покупки и количество
        (как конструктор, ещё повод задуматься о целесообразности такого конструктора)
        Метод смотрит, есть ли в массиве объектов Purchase такой товар и добавляет новый объект в массив
        или обновляет количество по существующей позиции
    5. Метод подсчета суммы товаров в корзине, принимающий мапу доступных к покупке товаров. Мапа используется
        только для получения из неё наименования товара, значит не нужна, ибо наименование есть в массиве объектов Purchase

2. Main
    1. Создаем Map список доступных к покупке товаров
    2. Ввод покупки пользователем
    3. Создаем пустой объект new Purchase
    4. Запускаем цикл
        - парсинг ввода
        - добавление данных в объект Purchase
